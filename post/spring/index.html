1.ApplicationContext的三个常用实现类：

classPathApplicationContext：

它可以加载类路径下的配置文件，要求配置文件必须在类路径下。（常用）

FileSystemXmlApplicationContext：

它可以加载磁盘任意路径下的配置文件。（必须有访问权）

AnnotationConfigApplicationContext：

它是用于读取注解创建容器

1.1ApplicationContext：

它在构建核心容器时，创建对象采取的是立即加载的方式

1.2BeanFactory

它在构建核心容器时，创建对象采取的是延迟加载的方式

2.Bean的作用范围

bean标签的scope属于可指定bean的作用范围：

singleton：单例（默认值）

prototype： 多例

request：作用于web应用的请求范围

session：作用于web应用的会话范围

global-session：作用于集群环境的会话范围（全局会话范围），当不是集群环境时，它等于session

3.2.Bean的生命范围

1.单例对象：

出生：当容器创建时对象出生

活着：容器存在。对象就一直存在

死亡：容器销毁，对象死亡

2.多例对象

出生：对象被使用时由Spring框架创建

活着：对象只要被使用就一直活着

死亡：当对象长时间不用，且没有对象引用时，由java的垃圾回收器回收

3.超简易BeanFactory

pro.properties:
	userDao=dao.impl.UserDaoImpl
	userService=service.impl.UserServiceImpl

public class BeanFactory {
    //配置文件
    private static Properties props;
    //容器
    private static Map<String, Object> beans;

    static {
        //加载配置文件
        props = new Properties();
        InputStream is = BeanFactory.class.getClassLoader().getResourceAsStream("pro.properties");
        try {
            props.load(is);
            beans = new HashMap<>();
            //遍历文件中所有的key
            Enumeration<Object> keys = props.keys();
            while (keys.hasMoreElements()) {
                String key = keys.nextElement().toString();
                String beanPath = props.getProperty(key);
                //为每个key所对应的的value创建对象
                Object value = Class.forName(beanPath).newInstance();
                //挨个放入容器中
                beans.put(key,value);
            }
        } catch (Exception e) {
            throw new ExceptionInInitializerError();
        }
    }

    public static Object getbean(String beanName){
        return beans.get(beanName);
    }
}
